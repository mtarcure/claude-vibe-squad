# Structured Data Authoring

Generate and validate JSON-LD / schema.org structured data that matches the page's visible content.

## Steps
1. Choose the applicable schema.org type(s) for the content (Article, Product, FAQ, etc.).
2. Populate required properties, then recommended ones; use real values from the page.
3. Emit valid JSON-LD.
4. Validate against the schema (required-property and type checks).
5. Confirm the markup matches the visible on-page content — no cloaking or spec-invented fields.

## Acceptance
- The JSON is valid and uses the correct type with required properties present.
- Markup reflects visible content (no cloaking).
- No invalid or spammy schema.
